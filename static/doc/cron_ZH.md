# ⏰ 定时任务推送平台配置文档

本文档详细说明了在不同消息平台配置定时任务主动推送消息时所需的关键标识和支持情况。

## 🚀 支持主动推送的平台及配置方式

![image](https://github.com/user-attachments/assets/ddedb6fd-ea6c-4e48-91b5-256d5bdcd232)


以下平台支持通过定时任务主动推送消息，并列出了所需的配置标识：

| 平台                     | 个人消息配置项           | 群组/频道消息配置项               | 备注                  |
|:-----------------------|:------------------|:-------------------------|:--------------------|
| **Telegram**           | 填写 **`个人userId`** | 填写 **`群`** 标识（通常是ChatID） |                     |
| **WeChat (微信)**        | 填写 **`个人userId`** | **不支持群组推送**              |                     |
| **Personal QQ (个人QQ)** | 填写 **`个人QQ号`**    | 填写 **`群号`**              |                     |
| **Lark (飞书)**          | 填写 **`chatId`**   | 需要 **`chatId`**          | 飞书通常使用ChatID作为会话标识。 |
| **Slack**              | 填写 **`chatId`**   | 需要 **`chatId`**          |                     |
| **Com WeChat (企业微信)**  | 填写 **`个人userId`** | **不支持群组推送**              |                     |

---

## 🚫 不支持主动推送的平台

以下平台**不支持**通过定时任务主动推送消息，或有特定的限制：

| 平台            | 不支持原因/限制说明                          |
|:--------------|:------------------------------------|
| **QQ**        | **不支持** 主动推送消息的功能。                  |
| **Discord**   | **不支持** 主动发送消息。只有在用户主动发送消息后，系统才能回复。 |
| **Ding (钉钉)** | **不支持** 通过回调处理的。                    |

---

## 📝 关键配置项说明

* **`个人userId` / `个人QQ号`**: 用于唯一标识一个**用户个人**账号或会话。
* **`群` / `群号`**: 用于唯一标识一个**群组**或群聊。
* **`chatId`**: 通常是飞书和 Slack 等平台用于唯一标识一个**对话/频道**的ID。

请根据您使用的平台和目标接收者（个人或群组）正确填写对应的标识符。
